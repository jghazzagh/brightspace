How to use this with D2L Brightspace

  1. Choose a page purpose and layout.
  2. Edit the page text, reusable cards/steps, checklist items, links, video details, colors, font size, and font stack.
  3. Review the accessibility checks. Warnings do not always mean the page is unusable, but they should be reviewed before copying.
  4. Open the Brightspace code tab and copy the generated code.
  5. In Brightspace, create or edit a content page.
  6. Open the HTML/source editor, then paste the generated code.
  7. Save, close, and reopen the page to confirm Brightspace did not strip or alter the layout.

Important notes

The file you are using on GitHub can include normal page-level CSS and JavaScript because it is the builder app. The copied output is different. The copied Brightspace code is designed to include only body-safe HTML with inline CSS. When font application is set to all generated text, the selected font is added inline to common text elements so Brightspace is less likely to override it.

For complex edits, it is usually cleaner to return to this builder and regenerate the page instead of heavily editing nested layout code inside the Brightspace visual editor.
